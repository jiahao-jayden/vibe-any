`src/config/core/website-config.tsx` centralizes core settings used across the app.

## Structure

```ts
export const websiteConfig = {
  metadata: {
    /* site title, theme, mode, images, social */
  },
  i18n: {
    /* default locale and supported locales */
  },
  monitoring: {
    /* computed from env: GA & Rollbar availability */
  },
  blog: {
    /* blog pagination etc. */
  },
  payment: {
    /* provider & credit packages */
  },
  planWithPrice: [
    /* plans and Stripe price IDs */
  ],
}
```

### Metadata & Social Links

| Field | Type | Purpose | Example |
| --- | --- | --- | --- |
| `theme.defaultTheme` | `'default'\| 'blue'\| 'green'\| 'amber'\| 'neutral'` | Default UI color theme | `'default'` |
| `theme.enableSwitch` | `boolean` | Allow users to switch theme | `true` |
| `mode.defaultMode` | `'system'\| 'light'\| 'dark'` | Default light/dark mode; `system` follows OS | `'system'` |
| `mode.enableSwitch` | `boolean` | Allow users to toggle light/dark | `true` |
| `images.ogImage` | `string` | Open Graph image path (under `/public`) | `'/og.avif'` |
| `images.logoLight` | `string` | Logo for light mode | `'/logo.avif'` |
| `images.logoDark` | `string` | Logo for dark mode | `'/logo-dark.avif'` |
| `social.github` | `string` | GitHub link | `'https://github.com/VibeAny'` |
| `social.twitter` | `string` | X(Twitter) link | `'https://x.com/VibeAny'` |
| `social.discord` | `string` | Discord invite link | `'https://discord.gg/xxxxxx'` |
| `social.youtube` | `string` | YouTube channel | `'https://www.youtube.com/@VibeAny'` |

Example:

```ts
metadata: {
  theme: { defaultTheme: 'default', enableSwitch: true },
  mode: { defaultMode: 'system', enableSwitch: true },
  images: { ogImage: '/og.avif', logoLight: '/logo.avif', logoDark: '/logo-dark.avif' },
  social: {
    github: 'https://github.com/VibeAny',
    twitter: 'https://x.com/VibeAny',
    discord: 'https://discord.gg/your-code',
    youtube: 'https://www.youtube.com/@VibeAny',
  },
}
```

### Internationalization (i18n)

| Field | Type | Purpose | Example |
| --- | --- | --- | --- |
| `defaultLocale` | `'en'\| 'zh'` | Default language | `'en'` |
| `locales` | `{ en: { name: string }, zh: { name: string } }` | Supported locales with display names | `{ en: { name: 'English' }, zh: { name: '中文' } }` |


### Monitoring

Monitoring includes analytics (GA) and error reporting (Rollbar). Both are controlled by env vars, and combined into derived switches.

Environment variables:

| Env | Meaning |
| --- | --- |
| `NEXT_PUBLIC_GA_ID` | Enable Google Analytics when set |
| `NEXT_PUBLIC_ROLLBAR_ACCESS_TOKEN` | Enable Rollbar error reporting when set |
| `NEXT_PUBLIC_ENABLE_GA_IN_DEV` | Enable GA in dev when `'true'` |
| `NEXT_PUBLIC_ENABLE_ERROR_REPORTING_IN_DEV` | Enable error reporting in dev when `'true'` |

Derived switches (`websiteConfig.monitoring`):

| Field | Type | Purpose |
| --- | --- | --- |
| `enableGoogleAnalytics` | `boolean` | Whether GA is enabled |
| `enableErrorReporting` | `boolean` | Whether Rollbar is enabled |

<Callout type="info">Production: tokens alone enable services; Development: also set the `NEXT_PUBLIC_ENABLE_*_IN_DEV` flags.</Callout>

> Use `shouldEnableGoogleAnalytics()` / `shouldEnableErrorReporting()` to handle dev vs prod.

### Payment

| Field | Type | Purpose | Example |
| --- | --- | --- | --- |
| `payment.enabled` | `boolean` | Toggle payment features | `true` |
| `payment.provider` | `'stripe'` | Payment provider | `'stripe'` |
| `payment.credit.enabled` | `boolean` | Enable credits system | `false` |
| `payment.credit.allowFreeUserPurchaseCredits` | `boolean` | Allow free users to purchase credit packages | `false` |
| `payment.credit.signupBonusCredits.enabled` | `boolean` | Enable signup bonus credits | `false` |
| `payment.credit.signupBonusCredits.amount` | `number` | Bonus credits amount | `0` |
| `payment.credit.signupBonusCredits.expireDays` | `number` | Expiry in days | `0` |

#### Provider env mapping

- Stripe
  - Keys: `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`, `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`
  - Prices: `NEXT_PUBLIC_STRIPE_PRO_MONTHLY_PRICE_ID`, `NEXT_PUBLIC_STRIPE_PRO_YEARLY_PRICE_ID`, `NEXT_PUBLIC_STRIPE_LIFETIME_PRICE_ID`, `NEXT_PUBLIC_STRIPE_100_CREDITS_PRICE_ID`, `NEXT_PUBLIC_STRIPE_200_CREDITS_PRICE_ID`
- Creem (optional)
  - Keys: `CREEM_X_API_KEY`, `CREEM_TEST_MODE`, `CREEM_WEBHOOK_SECRET`
  - Products: `NEXT_PUBLIC_CREEM_PRO_MONTHLY_PRODUCT_ID`, `NEXT_PUBLIC_CREEM_LIFETIME_PRODUCT_ID`, `NEXT_PUBLIC_CREEM_100_CREDITS_PRODUCT_ID`, `NEXT_PUBLIC_CREEM_200_CREDITS_PRODUCT_ID`

### Plans (`planWithPrice`)

Three plans are predefined and can be customized:

- `free`: no prices
- `pro`: two subscription prices (monthly and yearly)
- `lifetime`: one-time price

Each price requires a Stripe price ID from env:

| Plan | Env |
| ---- | --- |
| Pro Monthly | `NEXT_PUBLIC_STRIPE_PRO_MONTHLY_PRICE_ID` |
| Pro Yearly | `NEXT_PUBLIC_STRIPE_PRO_YEARLY_PRICE_ID` |
| Lifetime | `NEXT_PUBLIC_STRIPE_LIFETIME_PRICE_ID` |

Structure (example: `pro`):

| Field | Type | Description | Example |
| --- | --- | --- | --- |
| `id` | `string` | Plan ID | `'pro'` |
| `planType` | `'subscription'\| 'one_time'\| 'lifetime'` | Plan type | `'subscription'` |
| `credit.amount` | `number` | Credits per period | `100` |
| `credit.expireDays` | `number` | Credits expiry (days) | `31` |
| `prices[]` | `Array` | Price list | see below |

Each item in `prices[]`:

| Field | Type | Description | Example |
| --- | --- | --- | --- |
| `type` | `'subscription'\| 'one_time'` | Price type | `'subscription'` |
| `priceId` | `string` | Stripe Price ID | `process.env.NEXT_PUBLIC_STRIPE_PRO_MONTHLY_PRICE_ID!` |
| `amount` | `number` | Amount (integer) | `990` |
| `currency` | `string` | Currency | `'USD'` |
| `interval` | `'month'\| 'year'` | Billing interval (for subscription) | `'month'` |
### Credit Packages (optional)

If defined under `payment.credit.packages`, they will be exposed as one-time purchasable credit bundles. Each package requires:

- `id`: string
- `credit.amount`, `credit.expireDays`
- `price.priceId`, `price.amount`, `price.currency`

Use helpers in `config/features/payment-config.tsx` to transform packages into displayable plans.
