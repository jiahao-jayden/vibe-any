# Landing 页面系统

一个由可组合区块构成、具备完整国际化能力的生产级着陆页。通过 `getLandingPageConfig()` 拉取配置，在首页路由中按需渲染。

<Callout type="info">
每个区块都不是必填项。只要在文案/配置里提供该区块所需的最小字段，就会显示；不提供则不会显示（等同于关闭）。
</Callout>

## 配置与使用

<div className="fd-steps">
  <div className="fd-step">
    ### 1）准备 i18n 文案
    在各语言的 messages 文件中新增一个顶级键 `landing`，用于存放着陆页的文案与开关（省略某个区块的字段就会隐藏该区块）。

    ```jsonc title="messages（节选）"
    {
      "landing": {
        "banner": {
          "title": "VibeAny 1.0",
          "description": "用现成区块更快上线",
          "buttonText": "GitHub Star",
          "buttonHref": "https://github.com/...",
          "show": true,
          "dismissible": true,
          "variant": "gradient",
          "year": "2025"
        }
      }
    }
    ```
  </div>
  <div className="fd-step">
    ### 2）配置合作伙伴（PowerBy）
    在 `getPowerByConfig()` 中，把合作品牌的键名与 `public/partners/` 目录下同名的 SVG 文件对应起来（文件名需一致），并确保 SVG 的 `viewBox` 与尺寸统一。
  </div>
  <div className="fd-step">
    ### 3）可选：定价
    配置价格计划与相关环境变量，`getPricingConfig(locale)` 会将文案与计划合并后渲染。
  </div>
  <div className="fd-step">
    ### 4）在首页渲染
    使用配置方法并在存在数据时渲染对应区块。

    ```tsx title="src/app/[locale]/(site)/(home)/page.tsx（节选）"
    const { locale } = await params
    const [page, pricing] = await Promise.all([
      getLandingPageConfig(locale),
      getPricingConfig(locale),
    ])
    const powerBy = getPowerByConfig()

    return (
      <>
        {page.hero && <Hero hero={page.hero} />}
        {page.powerby && <PowerBy section={page.powerby} config={powerBy} />}
        {pricing && <Pricing section={pricing} />}
      </>
    )
    ```
  </div>
  <div className="fd-step">
    ### 5）预览与发布
    本地 `bun run dev` 预览；修改文案或伙伴图标仅需提交代码，无需改动页面逻辑。
  </div>
</div>

## 最佳实践

- 文案集中在 i18n，便于多语言维护。
- 使用 AVIF/WebP 等现代格式，并控制图片体积以提升性能。
- 新增合作伙伴时保持 SVG 尺寸与视窗统一，视觉更平衡。
- 在主要 CTA 上添加分析事件，衡量转化。

## 常见问题 Q&A

**Q: 首页某个区块不显示？**  
**A:** 该区块的文案可能在 `landing` 命名空间缺少必要字段，或在配置中被关闭（例如 banner 的 `show` 为 `false`）。只要提供最小字段即可显示对应组件 (＾_＾)。

**Q: PowerBy 的图标不出现？**  
**A:** 请确保 `getPowerByConfig()` 中的 `name` 能在 `public/partners/` 找到同名 SVG，并且 SVG 的 `viewBox`/尺寸一致，避免拉伸 (・ω・)。

**Q: Pricing 如何与支付计划联动？**  
**A:** `getPricingConfig(locale)` 会从价格计划配置中读取 plan 列表并与翻译合并；确保相关环境变量与定价 ID 已配置，然后 UI 会自动渲染 (・∀・)。

**Q: 怎样接入埋点？**  
**A:** 在按钮或关键交互处使用 `useAnalyticsEvents()`（例如 `events.click('hero_button', 'landing_page')`）。对于转化，可以在「开始使用」按钮点击后记录一次，或在注册/支付成功后记录更准确的转化 (￣ー￣)。

**Q: 多语言怎么切换？**  
**A:** 文案来自 next-intl，修改对应语言的 `landing` 命名空间即可。若某语言不需要某个区块，直接省略该区块的数据即可隐藏 (＾▽＾)。
