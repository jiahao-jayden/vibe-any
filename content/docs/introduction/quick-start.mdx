# VibeAny Guide

VibeAny is a modern AI scaffold built on Next.js. It ships user auth, AI integrations, payments, and an admin dashboard so you can ship ideas faster.

## Prerequisites (prepare these first)

<div className="fd-steps">
  <div className="fd-step">
    ### 1) Node.js
    A runtime for running JavaScript on your computer.
    If not installed, download the LTS version from the [official site](https://nodejs.org/).

    ```bash
    node --version
    ```
  </div>
  <div className="fd-step">
    ### 2) Git
    A version control tool for tracking history and collaborating.
    If not installed, get it from the [official site](https://git-scm.com/).

    ```bash
    git --version
    ```
  </div>
  <div className="fd-step">
    ### 3) Package manager
    Used to install dependencies. We recommend <code>bu n</code>, but <code>npm</code> or <code>pnpm</code> or <code>yarn</code> also work.

    ```bash
    # Install bun via npm if you haven't
    npm install -g bun

    # Check version
    bun --version
    ```
  </div>
</div>

## Get started (3 steps)

<div className="fd-steps">
  <div className="fd-step" >
    ### 1) Clone the repository
    <p>Download the code locally. You can rename <code>vibe-any</code> to any folder name.</p>
    <p>Prefer not to use Git? Download the ZIP directly:</p>
    <p>
      <a href="https://github.com/待修改/main.zip" target="_blank" rel="noreferrer">Download ZIP</a>
    </p>

    ```bash
    git clone https://github.com/待修改/vibe-any.git
    cd vibe-any
    ```
  </div>
  <div className="fd-step" >
    ### 2) Install dependencies
    <p>Pick one command below:</p>

    ```bash
    bun install
    npm install
    pnpm install
    yarn install
    ```
  </div>
  <div className="fd-step" >
    ### 3) Set environment variables
    <p><code>.env</code> stores the "keys" your app needs (database URL, OAuth keys, payment secrets). Copy the example file and fill in values.</p>

    ```bash
    cp .env.example .env
    ```
  </div>
</div>

> Tip: Common variables include <code>DATABASE_URL</code>, OAuth (GitHub/Google) configs, and payment secrets. See the Environment Variables doc for details.
